---
description: Issue から実装タスクファイルを作成する（勉強用・公式ドキュメント付き）
---

指定されたIssueファイルを読み込み、実装タスクファイルを作成してください。

## 要件

1. **学習目的のタスクファイル作成**
   - ユーザーが自分で実装できるように、詳細な手順を記載
   - 各タスクごとにセクションを分ける
   - 実装例は具体的かつコピペ可能な形式で提供

2. **公式ドキュメントの必須記載**
   - 実装例を記述する場合は、必ず公式ドキュメントまたは参考コードのURLを付ける
   - 各タスクセクションに「📚 公式ドキュメント」セクションを設ける
   - 複数のドキュメントがある場合はカテゴリ分けして記載

3. **ファイル命名規則**
   - ファイル名: `docs/{step-name}-tasks.md`
   - 例: `docs/step1-tasks.md`, `docs/league-api-tasks.md`
   - 作業内容が分かりやすい命名にする

4. **ファイル構成**
   ```markdown
   # {ステップ名}: {タイトル} - 実装タスク

   {簡単な説明}

   ---

   ## タスク1: {タスク名}

   ### ファイル: `{ファイルパス}`

   ### 実装内容

   {説明}

   ### 必要なパッケージ (該当する場合)

   ```bash
   bun add {package}
   ```

   ### 実装例

   ```typescript
   // コード例
   ```

   ### 実装のポイント

   1. **{ポイント1}**
      - 説明

   2. **{ポイント2}**
      - 説明

   ### テスト方法

   {テスト手順}

   ### 📚 公式ドキュメント

   **{カテゴリ1}:**
   - [{タイトル}]({URL})
   - [{タイトル}]({URL})

   **{カテゴリ2}:**
   - [{タイトル}]({URL})

   **重要な注意点:**
   - {ポイント}

   ---

   ## タスク2: ...

   ---

   ## {ステップ名}完了チェックリスト

   - [ ] タスク1が完了
   - [ ] タスク2が完了
   - [ ] `bun run lint` が通る

   ---

   ## 次のステップへ

   {次のステップの説明}

   ---

   **作成日:** {日付}
   ```

5. **WebSearchの活用**
   - 実装例に関連する公式ドキュメントを必ずWebSearchで検索する
   - 2025年の最新情報を取得する
   - 複数の信頼できるソースを提示する

## 実行フロー

1. ユーザーが指定したIssueファイルを読み込む
2. Issueの内容を解析し、タスクを抽出
3. 各タスクに必要な技術スタックを特定
4. WebSearchで公式ドキュメントを検索
5. タスクファイルを作成（上記フォーマットに従う）
6. ファイルパスをユーザーに報告

## 使用例

```
/create-tasks @docs/issues/issue-06-implement-league-api.md
```

上記の例では `docs/step1-tasks.md` のようなファイルを作成します。
